Lab Sheet 28: Cheapest Path (Dijkstra) — SmartRoute Deep Dive

1. Explanation
Finding the most cost-effective path to an LPG station is a classic optimization problem. In standard routing (like Google Maps), the goal is simply finding the shortest travel time. However, for auto drivers seeking LPG in Bangalore, the optimal destination depends on multiple changing factors.

In our SmartRoute system, Dijkstra's Algorithm systematically explores the "cheapest" known frontier based on a unique 3-Factor Composite Cost, ensuring that the first time we select a station, it is via the minimum possible overall cost to the driver's daily earnings.

2. The 3-Factor Composite Cost
Our algorithm computes a single composite cost (C) for every LPG station, merging three independent variables into a single comparable metric:
C = (α × travelTime) + (β × waitTime) + (γ × normalizedPrice × 40)

• Alpha (α) — Travel Weight: Controls how much distance matters. Raised when low on fuel.
• Beta (β) — Queue Weight: Controls how aggressively to avoid busy stations. Crucial during peak hours (11am-1pm).
• Gamma (γ) — Price Weight: Factors in LPG price (normalized and scaled). Vital for high-volume drivers maximizing monthly margins.

3. Algorithm Details
1. Create a priority queue (MinHeap) and insert the auto driver's current location with a cost of 0.
2. Maintain a map of the minimum cost discovered for each LPG station.
3. While the queue is not empty, extract the station with the lowest cumulative composite cost.
4. Dynamic Pruning: If a station's expected wait exceeds `maxWait` or travel distance exceeds `maxDist`, immediately prune (skip) it. It is considered infeasible regardless of price.
5. If the new cost is lower than previously recorded, update the map and push to the queue.

4. Dynamic Rerouting & IoT Simulation Integration
The algorithm doesn't just run once. Our IoT simulation continuously updates queue counts every 2 seconds based on virtual (and real, via GPS) driver activity.
Every time your target station's cost increases by more than the reroute threshold (e.g., 10%), Dijkstra runs again against the live data. If a better option emerges, a reroute alert fires, complete with Minimax validation to ensure the new route won't worsen before arrival.

5. Input Format
• Live GPS Coordinates: Driver's current location.
• IoT Real-time Feed: Array of 18 Bangalore LPG stations containing live queue counts, current price (₹/kg), and geographic coordinates.
• Constraints & Weights: User-defined or dynamic α, β, γ, maxWait, and maxDist.

6. Output Format
• Ranked list of feasible stations (not pruned).
• The single globally optimal (minimum total composite cost) LPG station to route to.

7. Python Code
Instead of raw nodes and edges, we implement this greedy strategy applied to our live LPG station network:

```python
import heapq

def find_cheapest_lpg_station(driver_loc, stations, alpha, beta, gamma, max_wait, max_dist):
    # Priority Queue stores: (composite_cost, station_id)
    pq = [(0, driver_loc)]
    min_costs = {driver_loc: 0}
    best_station = None

    while pq:
        current_cost, u = heapq.heappop(pq)

        # In our graph, the driver connects to all feasible stations
        for station in stations:
            travel_time = calculate_travel_time(u, station)
            expected_wait = station.expected_wait_time()

            # Dynamic Pruning: Skip infeasible stations immediately
            if expected_wait > max_wait or travel_time > max_dist:
                continue

            travel_cost = alpha * travel_time
            wait_cost = beta * expected_wait
            price_cost = gamma * normalize_price(station.price) * 40  # Scaled to match time units

            new_cost = current_cost + travel_cost + wait_cost + price_cost

            if station.id not in min_costs or new_cost < min_costs[station.id]:
                min_costs[station.id] = new_cost
                heapq.heappush(pq, (new_cost, station.id))

                # Keep track of the globally cheapest valid station destination
                if best_station is None or new_cost < min_costs[best_station]:
                    best_station = station.id

    return best_station, min_costs.get(best_station, -1)

# Helper functions for demonstration
def calculate_travel_time(src, dest): return 15  # Calculates Haversine distance * avg speed
def normalize_price(price): return (price - 70.0) / 10.0 # Maps price variations to a 0-1 scale
```

8. Why This Optimized Code Instead of Simple Code
A simple Breadth-First Search (BFS) only works for unweighted graphs. For complex LPG routing, a greedy approach using a Priority Queue prevents exploring redundant, expensive stations. Combined with Dynamic Pruning, it drastically reduces the search space, allowing the engine to run locally in the browser/app in under 20ms.
